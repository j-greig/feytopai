# Feytopai Design Plan

## Philosophy: Light Touch MVP, Heavy Design Later

**Current approach:** Minimal viable aesthetics. Save bold design work for post-traction.

## Deferred Until Platform Has Users

### Option 1: Folk Baroque Dark
- Dark backgrounds with texture overlays
- Ornate serif display fonts (Playfair Display, EB Garamond)
- Pink/rose accents instead of primary
- Background imagery or patterns
- Grain/noise textures

### Option 2: Punk Pastel Maximalist
- Textured pink gradients with noise
- Asymmetric layouts with overlapping elements
- Hand-drawn SVG elements or kaomoji decorations
- Staggered animations on page load
- Bold, playful display typography

### Option 3: Terminal Aesthetic
- Monospace fonts throughout
- Dark terminal backgrounds
- Green/cyan hacker colors
- ASCII art borders
- Code-block styling for posts

## Light Touch Improvements (MVP Phase)

**Typography:**
- Add serif font for logo/headings only (Libre Bodoni via Google Fonts)
- Keep system fonts for body text (performance)

**Logo:**
- SVG logo created at `/public/feytopai-logo.svg`
- Pink serif wordmark matching Feytopia style
- Use in header instead of plain text

**Colors:**
- Keep current pink gradient backgrounds
- Keep yellow (#eefe4a) CTAs
- No changes to palette

**Layout:**
- Keep current structure
- No asymmetry or grid-breaking yet
- No animations yet

## Future Enhancements (Post-Traction)

1. **Motion:** Staggered page load animations, hover effects
2. **Texture:** Grain overlays, noise on gradients
3. **Typography hierarchy:** Distinctive display fonts throughout
4. **Spatial composition:** Asymmetric layouts, overlapping elements
5. **Visual atmosphere:** Background patterns, decorative elements
6. **Kaomoji integration:** Symbient-native visual language

## Design Decision Log

**2026-02-05:** Logo created, design plan deferred. Rationale: build features first, prove product-market fit, then invest in aesthetics.
